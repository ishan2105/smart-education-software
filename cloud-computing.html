Elaborated Notes

1. Cloud Computing Fundamentals

  • Definition: Delivery of computing services over the internet.
  • Benefits: Scalability, cost-efficiency, flexibility, disaster recovery.
  • Key Concepts: On-demand self-service, broad network access, resource pooling, rapid elasticity, measured service.

2. Cloud Service Models

  • IaaS: Infrastructure as a Service (e.g., AWS EC2, Azure VM).
  • PaaS: Platform as a Service (e.g., Google App Engine, Azure App Service).
  • SaaS: Software as a Service (e.g., Gmail, Office 365).

3. Deployment Models

  • Public Cloud: Services offered over the public internet (e.g., AWS, Azure).
  • Private Cloud: Cloud infrastructure operated solely for one organization.
  • Hybrid Cloud: Combination of public and private clouds.

4. Virtualization & Containers

  • Virtualization: Creating virtual versions of servers, storage, networks.
  • Containers: Lightweight, portable, and consistent environments (e.g., Docker).

5. Popular Cloud Providers

  • AWS: Amazon Web Services – EC2, S3, Lambda, RDS, etc.
  • Azure: Microsoft Azure – Virtual Machines, Blob Storage, Functions, etc.
  • GCP: Google Cloud Platform – Compute Engine, Cloud Storage, BigQuery, etc.

6. Cloud Storage & Databases

  • Object Storage: AWS S3, Azure Blob, GCP Cloud Storage.
  • Databases: Managed SQL (RDS, Cloud SQL), NoSQL (DynamoDB, Cosmos DB).

7. Security in the Cloud

  • Identity & Access Management (IAM): Control user access and permissions.
  • Encryption: Data encryption at rest and in transit.
  • Compliance: GDPR, HIPAA, and other standards.

8. Serverless Computing

  • Concept: Run code without managing servers (e.g., AWS Lambda, Azure Functions).
  • Benefits: Automatic scaling, pay-per-use, simplified deployment.
